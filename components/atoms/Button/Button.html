7. 접근성 (Accessibility)

🎯 접근성 고려사항
  • 최소 터치 타겟: 모든 버튼은 최소 44×44px 크기 보장
  • 키보드 네비게이션: Tab, Enter, Space 키 지원
  • 포커스 인디케이터: 키보드 포커스 시 명확한 아웃라인
  • ARIA 레이블: 아이콘만 있는 버튼에 aria-label 필수
  • 색상 대비: 모든 변형이 WCAG AA 기준 충족
  • 상태 표시: disabled, loading 상태 명확히 구분
키보드 테스트 (Tab키로 이동해보세요):
<!-- 아이콘만 있는 버튼에 aria-label 필수 --> <button class="btn btn-primary btn-md btn-icon-only" aria-label="메뉴 열기"> <svg class="btn-icon" fill="currentColor" viewBox="0 0 20 20">...</svg> </button> <!-- disabled 상태 --> <button class="btn btn-primary btn-md" disabled>사용 불가</button> <!-- 로딩 상태 --> <button class="btn btn-primary btn-md btn-loading" aria-label="로딩 중">처리중...</button>

8. CSS 클래스 참조

/* 기본 구조 */ .btn // 기본 버튼 스타일 /* 변형 (Variants) */ .btn-primary // 보라색 주요 버튼 .btn-secondary // 녹색 보조 버튼 .btn-accent // 노란색 강조 버튼 .btn-outline // 테두리만 있는 버튼 .btn-ghost // 투명 배경 버튼 /* 크기 (Sizes) */ .btn-sm // 작음 (36px 높이) .btn-md // 보통 (44px 높이) - 기본값 .btn-lg // 큼 (52px 높이) .btn-xl // 매우 큼 (60px 높이) /* 상태 (States) */ .btn-loading // 로딩 상태 disabled // HTML disabled 속성 /* 아이콘 지원 */ .btn-icon // 아이콘 크기 조정 .btn-icon-left // 왼쪽 아이콘 마진 .btn-icon-right // 오른쪽 아이콘 마진 .btn-icon-only // 아이콘만 있는 원형 버튼 /* 조합 예시 */ .btn .btn-primary .btn-lg .btn-loading
